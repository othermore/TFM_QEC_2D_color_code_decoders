\chapter{Comparative analysis and discussion}
\label{chap:comparative_analysis}

After detailing the theoretical foundations of the 2D Color Code and implementing a comprehensive simulation pipeline, we now present the results of our empirical evaluation. In Chapter \ref{chap:simulating_2d_color_codes} we have already anticipated some comparisons  between the different decoders, exclusively in terms of threshold. This has allowed us to validate that the implementation was in line with the theoretical expectations and the existing published references. Now, we will conduct a deeper and wider comparison. 

We have simulated six different decoding algorithms under identical conditions: a 4.8.8 lattice geometry subjected to a code-capacity bit-flip noise model. The captured data allows measuring the four key metrics that will drive the comparative analysis:

\begin{itemize}
    \item \textbf{Determination of error thresholds ($p_{th}$):} Identifying the crossing point in the logical error curves where increasing the code distance begins to suppress errors rather than amplifying them.
    \item \textbf{Logical failure rate ($p_L$) scaling as a function of physical error rate ($p$) and code distance ($d$):} Analyzing the curves to evaluate the decoder's raw accuracy.
    \item \textbf{Sub-threshold scaling analysis and theoretical suppression limits:} Evaluating how steeply the logical error drops when operating below the threshold.
    \item \textbf{Computational complexity and empirical decoding time (Accuracy vs. Speed trade-off):} Analyzing the empirical decoding time to determine if the decoder's complexity is practically scalable.
\end{itemize}

\section{Determination of error thresholds (\texorpdfstring{$p_{th}$}{pTH}) for 4.8.8 color codes}

The error threshold represents the physical error rate below which the quantum error correction code becomes useful. To determine the threshold for each decoder, we evaluate the logical failure rate across multiple code distances ($d=5$ to $d=23$) and plot the results as a function of the physical error rate. 

\notatfm{As discussed in Chapter 6, the empirical threshold is estimated by finding the crossing point of the logical error curves for different distances. We compute the intersection of all pairs of curves and calculate the median of these crossing points.} Figure \ref{fig:7_thresholds} and Table \ref{tab:thresholds} show the calculated thresholds for all the evaluated decoders. While there are differences in their performing, the results suggest that the threshold will not be the main parameter to evaluate the capacity of a decoder, as they are all within a similar range.

\section{Logical failure rate (\texorpdfstring{$p_L$}{pL}) scaling as a function of physical error rate (\texorpdfstring{$p$}{p}) and code distance (\texorpdfstring{$d$}{d})}

While the threshold determines the absolute limit of the decoder, quantum computers will be useful when they can operate well below this threshold. It is crucial to evaluate how effectively a decoder suppresses logical errors in this operational regime. 

To visualize this, we introduce the concept of the mitigation ratio, calculated as $p_{phys} / p_L$. A value greater than 1 means the decoder is actively reducing the error compared to a single physical unencoded qubit. A higher ratio indicates a better performance.

Figure \ref{fig:7_mitigation_ratio} displays the mitigation ratio across various physical error rates for two fixed distances, $d=9$ and $d=23$. Additionally, Table \ref{tab:mitigation} summarizes these mitigation ratios for all decoders at a fixed physical error rate of $p=0.02$ (2\%), which is below the threshold of all evaluated algorithms and matches the lowest physical error rate we have been able to simulate.

The visualization reveals a critical scaling property. At a smaller distance ($d=9$), the BP+OSD decoder might appear a priori to be the most capable at mitigating noise in the low physical error regime, achieving a mitigation ratio of $\sim 8.2 \times 10^1$ compared to Möbius's $\sim 2.6 \times 10^1$. However, when we increase the topological protection to $d=23$, the Möbius decoder overtakes it and scales significantly better, achieving a mitigation ratio of $\sim 3.6 \times 10^4$ compared to BP+OSD's $\sim 1.6 \times 10^4$. Furthermore, the Projection decoder at the same distance achieves a mitigation ratio of just $\sim 5.2 \times 10^2$. This massive difference highlights that when operating below threshold and scaling the code, decoders that utilize the full topological structure without heuristic limitations (like Möbius) suppress errors exponentially faster.

\section{Sub-threshold scaling analysis and theoretical suppression limits}

\notatfm{In a log-linear plot (where the Y-axis is in logarithmic scale), this exponential decay manifests as a straight line. The parameter $A$ represents the slope, indicating how much the error is suppressed per unit of code distance, while the constant $C$ captures the baseline error overhead independent of the distance, shifting the line vertically to its correct height.} To predict the resources required to build a fault-tolerant quantum computer, and given that we are unable to run reliable simulations with physical noise levels below $p_{phys} = 2\%$, we must mathematically model how the logical error drops as we increase the size of the code. We modeled the logical error rate using an exponential decay function defined as $\log_{10}(p_L) = A \cdot d + C$. 

Using \texttt{SciPy}'s \texttt{curve\_fit} function, we performed a 2D surface regression to fit the empirical data across both variables simultaneously (code distance $d$ and physical error rate $p$). The full surface model equation is $\log_{10}(p_L) = c_0 + c_1 d + c_2 \log_{10} p + c_3 d \log_{10} p$. The parameters resulting from this surface regression for all decoders, as well as their $R^2$ coefficients that indicate the goodness of the fit, are documented in Table \ref{tab:error_scaling_fit}. 

To visualize this fit, Figure \ref{fig:7_error_scaling} shows a one-dimensional "slice" of this 2D mathematical surface. By fixing the physical error rate at $p=0.02$, the complex surface equation simplifies into a straight 1D line ($\log_{10}(p_L) = A \cdot d + C$), which we plot alongside the empirical data points for that specific noise level.

The regressions confirm a strictly exponential suppression of errors. The slope of these lines represents the suppression efficiency of the algorithm. Möbius displays the steepest slope, meaning it requires adding fewer physical qubits to gain an extra order of magnitude in logical protection.

Using these mathematical fits, we extrapolated the data to answer a practical engineering question: What code distance is required to achieve a target logical error rate? We defined a strict target logical error rate of $p_L = 10^{-12}$ (a standard requirement for large-scale algorithm execution), and evaluated the required distance under two different baseline physical error rates: $p = 10^{-3}$ and $p = 10^{-4}$.

The Y-axis shows exclusively valid odd distances.}
    \label{fig:7_required_distances}

\notatfm{$N = \frac{3}{4}d^2 + \frac{1}{4}$ calculates exclusively the number of \textit{data} qubits in the triangular color code lattice. The total physical hardware footprint will be at least double, as it must also include the ancilla (measurement) qubits interspersed in the lattice to extract the error syndromes.} As shown in Figure \ref{fig:7_required_distances}, the choice of decoder drastically affects the hardware requirements. To achieve $p_L=10^{-12}$ with a base physical error rate of $10^{-3}$, the Möbius decoder only requires a code distance of $d=15$. In stark contrast, the Concatenated MWPM and the Projection decoders require $d=23$ and $d=21$, respectively. Given that the number of physical qubits in a 2D color code scales quadratically with the distance ($N = \frac{d^2 + 2d - 1}{2}$), moving from $d=15$ (127 qubits) to $d=23$ (287 qubits) more than doubles the size of the required quantum processor, highlighting the immense value of a highly accurate decoder. Furthermore, if the hardware quality improves to a baseline physical error rate of $10^{-4}$, the required distances drop drastically across all decoders, with BP+OSD and Möbius requiring only $d=9$ (49 qubits) and $d=11$ (71 qubits), respectively, to achieve the exact same $10^{-12}$ logical target.

\section{Computational complexity and empirical decoding time}

The final dimension of our comparative analysis focuses on execution time. A decoder can have the necessary accuracy for a given problem, but if it is too slow, errors will accumulate in the quantum processor while the classical computer tries to calculate the correction.

\notatfm{In a log-log plot, a pure polynomial scaling ($A \cdot N^k$) would appear as a straight line. However, we introduced the constant $C$ to capture the fixed software overhead that dominates at small distances, creating the curved shapes seen in the plot.} Using, once again, \texttt{SciPy}'s \texttt{curve\_fit} function, we analyzed the average execution time for each decoder. The empirical data was fitted to a complexity model $T(N) = C + A \cdot N^k$, where $N$ is the number of qubits and $k$ is the scaling exponent. The resulting parameters are summarized in Table \ref{tab:time_scaling_fit}.

\notatfm{The 100x speedup assumption is based on the known existance of under micro-second implementations of MWPM\cite{Wu_2025} and $11.5 ns$ execution times for Union-Find on FPGAs\cite{Liyanage_2024}.} To provide a realistic hardware context, we tried to extrapolate these times to the required distances calculated in the previous section. Figure \ref{fig:7_required_times} shows the estimated decoding time required to reach the target logical error rates. We have assumed a 100x hardware acceleration factor to simulate the transition from our Python-based simulation environment to a dedicated FPGA implementation. While this is possibly a valid assumption for most of the decoders, the Möbius and BP+OSD decoders might require a lower acceleration factor due to their native C++ implementation.

The most critical element in Figure \ref{fig:7_required_times} is the green band at the bottom, which represents the typical time budget for a single cycle in a modern superconducting quantum processor (between $100 ns$ and $1 \mu s$ \cite{Google_Willow_2024}). If the decoder takes longer than this budget, it will not be able to decode real time, causing error accumulation or forcing the quantum computer to idle, increasing the chances of errors.

We observe a significant gap between the optimized C++ libraries and the custom Python implementations. The Python wrappers introduce a constant computational overhead per shot that heavily penalizes their absolute times. This explains the good performance of BP+OSD when compared with other python implementations like the Correlated or Concatenated MWPM decoders.

It is worth noting the slowness of the Concatenated MWPM decoder. While this decoder is expected to come with an overhead (as it implements twice as much MWPM matchings as others), the running time per shot is simply at a different scale. We believe that the actual implementation of the chosen library suffers some serious efficiency issue, perhaps executing certain calculations during the decode phase that could be cached if they only depend in the DEM.

Only a few decoders are realistic candidates to perform real-time error correction (note that this estimation is for a simplified code-capacity error model, and in a realistic noise model, significantly larger distances would be required). 

This analysis validates a fundamental conclusion: while achieving high accuracy (effective error suppression, resulting in lower required code distances) is critical to minimizing the number of physical qubits required and the complexity of the decoding, computational speed is equally paramount. To build a useful fault-tolerant quantum computer, the industry will require decoders that are both highly accurate and inherently parallelizable or capable of being executed entirely in specialized low-level hardware.

\section{Final evaluation}
While each decoder has unique characteristics, and the "best" decoder may depend on the specific application, a general assesment can be made. All decoders provide thresholds in the $7\%-9\%$ range, making the threshold itself not the most distinguishing feature. 

Because of its unique error mitigation and decoding time scaling properties, the Möbius decoder stands out as a very promising candidate for fault-tolerant quantum computing, although it seems to require a significant distance to deliver its full potential. 

While Concatenated MWPM and Correlated decoders are meant to produce improvements over the Projection and Restriction decoders, and these are visible as the code scale, the difference is not enough to reduce significantly the required code distance, but the decoding times suffer a penalty. 

Therefore, aside the Möbius decoder, only Restriction and Projection decoders would be good candidates to implement real-time error correction.
